# 旅行レビュースキル（travel-review）

## 目的
作成した旅行データとUIを「初めてその国に行く日本人旅行者」の目線でレビューし、不足・不備を洗い出す。

## レビュアーの立場
以下の人物になりきってレビューする:
- 東京在住、インドネシアは初めて
- このサイトだけを頼りに実際に旅行する
- 英語は少しできるが、インドネシア語はまったくわからない
- スマホ1台で行動する

## レビューチェックリスト

### Level 1: 移動・アクセス（最重要）
- [ ] 前のスポットから次のスポットへの移動方法が**具体的に**書かれているか
- [ ] 「タクシーで行ける」だけでなく「Grabで〇〇と検索」レベルの具体性があるか
- [ ] 所要時間が明記されているか
- [ ] 費用（IDR + JPY）が明記されているか
- [ ] 初めてでも迷わない手順になっているか

### Level 2: 費用・金額
- [ ] すべてのスポットに入場料が書かれているか（無料なら「無料」と明記）
- [ ] 食事の予算感が分かるか
- [ ] 1日あたりの概算予算が出せる程度に金額が網羅されているか
- [ ] IDRと日本円の両方が併記されているか
- [ ] IDR→JPY換算レートは0.0094で統一されているか

### Level 3: 画像・ビジュアル
- [ ] 各スポットに画像があるか（最低1枚）
- [ ] 画像を見て「行ってみたい」と思えるか
- [ ] 食事の写真はあるか
- [ ] エリアの雰囲気が伝わるヒーロー画像はあるか

### Level 4: 安全・文化・実用（追加基準）
- [ ] 服装規定（dressCode）が必要なスポットに記載されているか
- [ ] 持ち物リスト（mustBring）が危険なスポットに記載されているか
- [ ] 文化的マナー（左手禁止、頭触らない等）が記載されているか
- [ ] 火山・海・滝の安全警告が適切か
- [ ] 健康証明書・QRIS等の最新規制が反映されているか
- [ ] 緊急時の対処法が記載されているか
- [ ] 季節（乾季/雨季）による影響が記載されているか

### Level 5: 食事情報
- [ ] 各エリアに食事情報があるか
- [ ] 予算帯別（安い/中/高い）の選択肢があるか
- [ ] 具体的な店名・場所が書いてあるか
- [ ] その地域の名物が紹介されているか
- [ ] 「これを食べろ」一品があるか

### Level 6: 実用性
- [ ] 曖昧な表現がないか（「近い」→何分？「安い」→いくら？）
- [ ] 「で、結局どうすればいいの？」と思う箇所がないか
- [ ] Google Mapsリンクがあるか
- [ ] 営業時間・定休日が書いてあるか
- [ ] 注意事項・Tipsが現実的か

### Level 7: 情報の信頼性
- [ ] 明らかに古い情報や誤った情報がないか
- [ ] 存在しない場所や閉業した店がないか
- [ ] 料金が相場から大きくかけ離れていないか
- [ ] 料金は2-3ソースで検証されているか

### Level 8: 整合性チェック
- [ ] タイムラインが1日に収まるか（移動時間＋滞在時間が24時間以内）
- [ ] 前日の transport.to と翌日の最初のtimelineが矛盾しないか
- [ ] 価格フォーマットが統一されているか（fee: {idr, jpy}）
- [ ] 都市名の表記が統一されているか

### Level 9: UI/UX
- [ ] スマホで見やすいか
- [ ] 旅行中に片手で操作できるか
- [ ] AI感がないか（「〜することをお勧めします」等の機械的表現）
- [ ] 旅行ガイドブックのような自然な文体か
- [ ] タップ・スクロールが直感的か

## レビュー結果の出力形式

```markdown
## レビュー結果: Day X - [エリア名]

### 致命的な問題（すぐ修正必須）
- [問題の内容と具体的な修正指示]

### 改善が必要（次のイテレーションで修正）
- [問題の内容と具体的な修正指示]

### 良い点
- [そのまま維持すべきポイント]

### 追加調査が必要
- [不足している情報と調査方法の提案]
```

## レビューサイクル
1. データレビュー → 修正 → 再レビュー（最低2回）
2. UIレビュー → 修正 → 再レビュー（最低2回）
3. 統合レビュー → 最終修正
